Infrastructure Documentation Standards
=====================================

Welcome to the Infrastructure Documentation Standards project. This system provides
standardized templates and automated tools for creating consistent, comprehensive
documentation for client infrastructure projects.

.. note::
   **Purpose**: Eliminate "Where is the config for X?" problems during maintenance
   and troubleshooting by establishing clear documentation standards and gap assessment processes.

Quick Start
----------

Choose the appropriate template for your project:

**For Client Handoffs**:
  Use the :doc:`_templates/transition-checklist` to ensure all critical information is available.

**For Azure Environments**:  
  Use the :doc:`_templates/azure-infrastructure` template for comprehensive infrastructure documentation.

**For Database Projects**:
  Use the :doc:`_templates/sql-database-guide` template for database-specific configuration and troubleshooting.

Automation Tools
---------------

This project includes automation tools for:

* **Template Generation**: Create project-specific documentation from templates
* **Gap Assessment**: Identify missing critical documentation
* **Azure Integration**: Auto-discover and document Azure resources
* **SQL Documentation**: Generate database schemas and connection documentation
* **ITGlue Integration**: Deploy documentation to your knowledge management system

Usage Examples
--------------

**Generate a new project from template**:

.. code-block:: bash

   docs-setup --template azure-infrastructure --client "Client Name" --project "Project Name"

**Build documentation**:

.. code-block:: bash

   docs-build --format html
   docs-build --format pdf  # For client deliverables

**Auto-discover Azure resources**:

.. code-block:: bash

   azure-docs --subscription "subscription-id" --resource-group "rg-name"

**Deploy to ITGlue**:

.. code-block:: bash

   deploy-itglue --client "Client Name" --format html
